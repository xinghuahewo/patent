# 一种自治系统注册地与运营地不一致的识别方法

所述方法以自治系统为分析对象，以 **AS-月** 为最小分析单位，基于行政分配信息、注册信息、静态地理信息、连接结构信息、主动路径测量信息及设施信息，识别注册地与运营地不一致的疑似离岸自治系统。
所述方法优选采用 **90 天边界网关协议观测数据、30 天路径测量数据和周级设施快照数据** 作为统一时间组织窗口。

## S1：获取自治系统的行政分配信息并构建历史分配归属序列

获取区域互联网注册管理机构发布的资源分配数据，从中提取目标自治系统对应的分配侧国家代码、所属区域互联网注册管理机构、分配日期及分配状态，并按时间顺序构建该自治系统的历史分配归属序列。
若同一自治系统在连续时间窗口中存在跨国分配归属变化，则将其标记为行政归属异常候选。

## S2：获取自治系统的注册对象信息并建立行政注册基线

获取与目标自治系统相关联的 RDAP 数据和/或 WHOIS 数据，解析自治系统对象、组织对象及上级实体对象的国家字段、组织名称及对象关系，得到注册对象侧国家；并将其与步骤 S1 得到的分配侧国家进行交叉比对，建立该自治系统的行政注册基线。
若分配侧国家与注册对象侧国家不一致，或存在明显多值国家冲突，则将该自治系统纳入行政冲突候选集合。

## S3：获取起源前缀并构建静态候选运营国家集合

获取目标自治系统在边界网关协议中起源宣告的路由前缀集合，对各路由前缀执行随机采样与边界采样，抽取代表性 IP 地址样本；利用多个公开 IP 地理位置数据库对所述代表性 IP 地址进行国家映射，并通过多库投票和覆盖比例统计确定各前缀的主导国家。
进一步汇总形成该自治系统的静态候选运营国家集合及主导前缀国家 DominantPrefixCountry。
若所述静态候选运营国家集合中存在异于注册国的国家，且对应前缀覆盖比例超过预设阈值并在连续分析窗口中稳定存在，则将该自治系统纳入静态地理冲突候选集合。

## S4：构建自治系统的月度连接结构画像

获取 RIPEstat 提供的 observed neighbours 数据以及 CAIDA AS Rank 提供的 provider、customer、peer 关系数据，对所述数据进行标准化处理，并按 `(asn, neighbor_asn, month)` 进行融合，形成目标自治系统的月度连接结构画像。
所述连接结构画像至少包括邻居数量、provider 数量、customer 数量、peer 数量、邻居新增数量、邻居消失数量、邻居波动率以及 provider 切换情况。

## S5：识别结构异常与边界型自治系统并构建第一阶段嫌疑集合

基于步骤 S4 得到的月度连接结构画像，识别长期仅依赖少数上游、邻居数显著偏低、连接波动率高、provider 频繁切换或连接模式显著偏离同类正常自治系统的结构异常候选；
进一步识别具有明显跨境依赖、国际互联或边缘出口特征的边界型自治系统。
将步骤 S2 得到的行政冲突候选、步骤 S3 得到的静态地理冲突候选以及本步骤得到的结构异常候选进行交叉分析、并集、去重和统一编号，形成第一阶段嫌疑自治系统集合。
其中，连接结构证据仅用于识别结构异常与边界型特征，不直接用于推断目标自治系统的运营国家。

## S6：对第一阶段嫌疑集合进行优先级分层并选取确认目标

根据第一阶段嫌疑自治系统集合中各目标的证据数量、证据强度及时间窗口持续性，对其划分为高优先级嫌疑集合、中优先级观察集合和低优先级背景集合；
优选从高优先级嫌疑集合及部分中优先级观察集合中选取目标自治系统，进入后续主动确认阶段。

## S7：基于多探针路径测量确认边界进入位置和稳定路径落点国家

从多个国家和多个自治系统视角选取测量探针，针对目标自治系统起源前缀中的代表性 IP 地址发起 traceroute 或 Paris traceroute 测量；对测量结果中的路径跃点进行解析，识别进入目标自治系统之前和之后的边界跃点；对边界接口 IP 进行命名规则解析、地理位置归因和多探针交叉验证，形成目标自治系统的边界进入国家 BorderIngressCountry 和稳定路径落点国家 LastStableCountry。
若多个探针和多个时点下的边界进入位置或稳定路径落点长期集中于异于注册国的国家，则将该异国作为路径确认侧候选运营国家。

## S8：基于设施与交换点数据确认物理落点国家并融合多源证据

获取目标自治系统在 PeeringDB、交换点信息源及设施信息源中的 presence 数据，按周级时间粒度保存设施快照，统计其在不同国家上的交换点出现次数、设施出现次数和连续存在时间，形成主导设施国家 DominantInfraCountry；
进一步融合分配侧国家、注册对象侧国家、DominantPrefixCountry、DominantInfraCountry、BorderIngressCountry 和 LastStableCountry，并结合连接结构摘要及行为异常指标，更新 GeoMismatchScore、MismatchScore 和 EvidenceSummary，形成目标自治系统的综合运营地画像。

## S9：进行人工复核并输出最终确认结果

针对进入最终确认阶段的自治系统，生成单 AS 案例卡，展示其分配侧国家、注册对象侧国家、主导前缀国家、主导设施国家、边界进入国家、稳定路径落点国家、连接结构摘要、评分结果及证据摘要；
由人工复核人员检查是否存在跨国集团、云服务商、内容分发网络、骨干网、设施公开信息不完整、路径测量异常或其他合理解释；
并根据复核结果将目标自治系统标记为正常边界型自治系统、可解释跨国运营自治系统、疑似离岸运营自治系统或证据不足待复核自治系统，从而输出最终确认结果。

## 可作为权利要求/说明书结尾的总括句

> 通过上述步骤，先基于行政分配信息、注册对象信息和静态地理映射信息形成第一阶段嫌疑自治系统集合，再通过连接结构分析、路径测量、设施核查、多源证据融合及人工复核，确认目标自治系统的主导前缀国家、主导设施国家和稳定路径落点国家，并据此区分正常边界型自治系统、可解释跨国运营自治系统与疑似离岸运营自治系统，从而实现对自治系统注册地与运营地不一致情形的识别。
