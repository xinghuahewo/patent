# 文档归档说明

本目录保存已被替代的旧状态、早期草稿和历史计划。归档文件可以作为背景参考，但不能作为当前执行入口。

当前有效入口在上一级目录：

- `docs/status.md`
- `docs/runbook.md`
- `docs/roadmap.md`
- `docs/schema.md`
- `docs/schemas/`
- `docs/artifacts.md`

当前仍有效的拆分 schema 已移到 `docs/schemas/`，不再放在 archive。

归档判断规则：

- 已被新文档完整替代的旧状态快照，可以归档。
- 一次性试跑记录、早期说明、旧计划，可以归档。
- 仍承担长期路线、字段契约、执行入口或维护规则的内容，不应归档；应移到上一级有效文档或 `docs/schemas/`。

当前 archive 中仍可用于追溯背景的文件：

- `tech_map.md`：已抽取有效长期路线到 `docs/roadmap.md`。
- `project_flow.md`：已抽取有效流程说明到 `docs/roadmap.md`、`docs/runbook.md` 和 `docs/status.md`。
- `patent_2.md`：作为 S1-S9 方法原稿保留，用于追溯专利步骤表述。
